Maybe we should have a conversation about it, because it is not super straightforward, and a case by case decision I think. But here are some context and principles:
- We need to be able to execute a "flip and stay" DR exercise in October. This is non negotiable, as we are contractually obligated to demonstrate this to Cardinal Health.
- Since this is non-negotiable, the thinking is that it is better to front-load the work, since it will have to be done soon enough anyway, and it brings clarity and removes uncertainty sooner. This clarity (i.e. a "plan") would otherwise would have to be achieved with spikes in the next few sprints anyway... so we might as well just start the work.
- With that said, if there is a better sequencing that results in less wasted effort, and it can be done within this timeframe (before October), that is definitely on the table. 
	- For example, if we can do the Aurora migration first, then the DR work on Aurora, and we are confident in accomplishing that before October, that seems like a better path.

So in other words, we can do whatever we want, and should do what makes most sense, as long as we are ready by October. However, if we have a better, less wasteful path in mind, but it puts October in jeopardy, we unfortunately have to forego this better path.

Makes sense?
